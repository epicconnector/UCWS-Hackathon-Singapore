## Project Structure

Organize your project folder with the following structure:

```
projects/
├── project-name/
│   ├── README.md              # Project overview and documentation
│   ├── src/                   # Source code
│   ├── docs/                  # Additional documentation
│   ├── requirements.txt       # Python dependencies
│   ├── package.json           # JavaScript dependencies
│   ├── .env.example           # Environment variables template
│   ├── LICENSE                # License file
│   └── demo/                  # Demo files, screenshots, videos
```

## What to Include

### Code
- Clean, well-commented code
- Proper file structure and organization
- Follow language-specific conventions
- Include configuration files (.env.example, config files)

- Comprehensive README.md

## Submission Process

### Step 1: Official Platform Submission
Submit your project through the official platform first:
👉 [evol.epicconnector.ai](https://evol.epicconnector.ai)

### Step 2: Repository Submission (Optional)
After platform submission, if you want to showcase on GitHub:

1. **Fork or Clone** this repository
2. **Create a new folder** under `projects/` with your project name
3. **Add your project files** following the structure above
4. **Write comprehensive documentation** (README, etc.)
5. **Test your setup instructions** to ensure they work
6. **Create a Pull Request** with your project

### Step 3: Review & Merge
- Organizers will review your submission
- Ensure your project meets guidelines
- Once approved, your project will be merged

## Quality Checklist

Before submitting, verify:

- ✅ Project follows the folder structure
- ✅ README.md is comprehensive and well-formatted
- ✅ Code is clean and well-commented
- ✅ Installation instructions are accurate and tested
- ✅ All dependencies are documented
- ✅ No sensitive information (API keys, passwords) in code
- ✅ License file is included
- ✅ .gitignore is properly configured
- ✅ Demo/screenshots are included
- ✅ Project name is descriptive and follows naming conventions

---

## Naming Conventions

- **Folder names:** Use lowercase with hyphens (e.g., `ai-chat-bot`, `smart-contracts`)
- **File names:** Use lowercase with hyphens or underscores (e.g., `config.py`, `auth_module.js`)
- **Branch names:** Use descriptive names (e.g., `feature/submission`, `docs/setup`)
